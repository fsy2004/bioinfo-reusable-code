## 筛选依据

参考 `C:\Users\fsy\Desktop\生信` 中论文的共同方向，当前最适合补进本代码库的不是重复的差异分析或富集脚本，而是这些新模块：

- 单细胞虚拟扰动：与已有 `scTenifoldKnk`、`scFOCAL` 脚本互补。
- 扰动数据资源：给虚拟扰动模型提供 Perturb-seq/CRISPR/single-cell perturbation 训练或验证数据。
- 药物扰动与药物重定位：对应论文里的环境毒理、药物靶点、治疗组合筛选。
- 空间转录组通讯与细胞命运：补强空间、多视角轨迹和细胞命运驱动基因分析。
- MR + 共定位：让“药物靶点/暴露-疾病”推断更接近因果证据链。
- TF/通路活性评分：把差异基因升级为可解释的调控活性、通路活性特征。

建议先不要直接复制外部仓库源码。优先采用“安装依赖 + 写本仓库包装脚本 + 记录版本/参数”的方式；确实需要整仓库代码时，再用 git submodule 或 vendor 目录，并先检查许可证。

## 我建议的接入顺序

1. 先接 `scPerturb/scperturbR`：最容易和 Seurat 连接，能马上为虚拟扰动提供外部扰动数据/统计检验。
2. 再接 `CellOracle`：补 GRN 级别的虚拟扰动解释，和已有 scTenifoldKnk/scFOCAL 最互补。
3. 接 `GEARS`：如果后续想做“多基因组合扰动预测”，它是更有新意的方向，但需要 Python/torch/pyg 环境。
4. 接 `chemCPA` 或 `scDrug`：选择一个药物扰动路线。想偏机器学习预测选 chemCPA；想快速出药物重定位报告选 scDrug。
5. 最后把 `TwoSampleMR + coloc` 做成标准靶点证据链：候选基因/蛋白 -> MR -> 共定位 -> 单细胞定位 -> 药物扰动。
