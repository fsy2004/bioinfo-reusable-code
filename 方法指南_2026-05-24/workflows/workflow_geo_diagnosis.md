# workflow_geo_diagnosis

用途：GEO 或 bulk RNA-seq 疾病诊断/机制分析路线。目标是把表达矩阵整理、差异分析、富集、WGCNA、机器学习、诊断验证和免疫解释串成一条可复用闭环。

## 适用场景

- 非肿瘤炎症、免疫、代谢、环境暴露相关疾病的 GEO 诊断模型。
- 多队列公共数据整合和外部验证。
- 以候选基因、特征签名或疾病微环境解释为核心的文章。

## 不适用场景

- 原始 count RNA-seq 且需要 DESeq2/edgeR 的项目：当前库里主流程仍偏 limma/芯片矩阵。
- 无外部验证队列的小样本项目：机器学习结果只能作为探索。
- 只有单细胞或空间数据的项目：优先进入单细胞/空间 workflow。

## 输入规范

| 输入 | 推荐格式 | 关键字段 |
|---|---|---|
| 表达矩阵 | CSV/TSV/TXT，行为基因、列为样本 | gene symbol 或 probe ID |
| 分组表 | CSV/TSV | sample、group、batch 可选 |
| 多队列表 | 每个队列一个矩阵 | 样本名唯一，基因 ID 可合并 |
| 候选基因 | TXT/CSV | gene |
| 外部验证集 | 表达矩阵+分组表 | 与训练集同一基因命名体系 |

差异基因发现 -> 通路富集 -> 共表达模块 -> 多算法机器学习 -> SHAP 解释 -> 外部队列诊断验证 -> 免疫浸润和单细胞定位。

## 质量控制检查

- 样本名在表达矩阵和分组表中完全一致。
- 基因 ID 去重策略记录清楚。
- 训练集和验证集不混用。
- ML 特征筛选不在全量数据上提前泄漏验证集信息。
- WGCNA 样本数不足时不要强行作为核心结论。
