# 分析质量自查清单 · 12 铁律

> 对应记忆 `feedback_analysis_quality`。每个分析项目：**开工前**过一遍「分析前」，**完成后**过一遍「出图与收尾」。
> 机检：`python _framework/qc_lint.py <你的脚本或目录>` 可自动查出常见违规。

## 一、分析前
- [ ] 查了该方法的旗舰论文 / 官方文档，确认**适用边界与假设**（不套模板）
- [ ] 核对工具版本与接口（Seurat v4↔v5 API、Scanpy 新版弃用函数）；工具选型匹配数据类型 〔铁律1〕
- [ ] `config` 里设了 `SEED`，并计划对每个随机步骤显式传种子 〔铁律1〕
- [ ] 路径全走 config/相对路径，**无 `setwd()`/绝对路径硬编码** 〔铁律5〕
- [ ] 先查 `_framework` 与代码库现成模块（`reference_code_library`），**复用不重写** 〔铁律5〕

## 二、分析中
- [ ] QC 阈值按**本数据集分布**定（先画 nFeature/nCount/mt% 分布），非通用 200/20% 〔铁律2〕
- [ ] 细胞/亚群注释做了 **marker 基因验证**，自动注释只作参考，可疑簇标 `unknown` 〔铁律2〕
- [ ] 关键参数（n_pcs / resolution 等）用**诊断图**核实（Elbow/clustree），非默认一把过 〔铁律1/3〕
- [ ] 结果过滤假阳性（校正 p + effect size + 最小细胞/样本数），每个结论有生物学支撑 〔铁律2〕
- [ ] 高级逻辑：空间-单细胞映射归一化统一 / 轨迹起点选型有据 / **虚拟扰动验证调控网络**（不只看表型）/ SHAP 按亚群**分层** 〔铁律3〕
- [ ] 大矩阵内存友好（稀疏/分块/及时 `gc()`）；耗时步骤用 `cache_step()` 断点续跑 〔铁律5〕

## 三、出图与收尾
- [ ] 出图用 `theme_pub()`+`pal_pub()`（**色盲友好**、同类细胞跨图**配色统一**），`save_fig()` 出**矢量 PDF+300dpi** 〔铁律4〕
- [ ] 图要素齐全：比例尺 / 统计标注 / 分组 / 图例 / 单位；图中文字**英文** 〔铁律4〕
- [ ] 关键统计值用 `log_stat()` 落盘，文稿数字从 `logs/` 取，**不手填** 〔铁律6〕
- [ ] `save_session()` 记录 `sessionInfo` / `pip freeze`（依赖版本锁定）〔铁律6〕
- [ ] 代码改动 git 留痕 + README 更新 〔铁律6〕
- [ ] **自我反思**：每步结果生物学合理吗？有无更优解？哪里可能错？（主动找问题，不等指出）
- [ ] 跑 `python _framework/qc_lint.py <脚本/目录>` 机检通过
